# UI Repro — スクリーンショットからUIを高精度に再現するスキル

スクリーンショットをもとにUIを高精度に再現するためのClaude Codeスキル。
「画像 → 言語化 → 実装 → 撮影 → スクショ比較で差分を言語化 → 修正」のループで再現精度を上げる。
言語化テキストが実装の唯一のインプットとなる。

## なぜ「言語化を介在させる」のか

Claude（LLM全般）は画像を直接見てコードを書くのが苦手。スクショを渡して「これ通りに作って」と言うと、大まかな構造は合うが細部の色・余白・フォントサイズ等がズレる。

原因は、画像からコードへの変換が一発で行われるため、視覚的な情報が中間で失われること。

このスキルでは画像を一旦テキスト（具体的な数値・色コード）に言語化し、そのテキストだけを見て実装する。差分は元スクショと現状スクショを並べて比較し、違いを言語化して抽出する。こうすることで:

- 曖昧な視覚判断が排除される
- 差分が構造化されて修正しやすくなる
- ループごとに確実に精度が上がる

## 使い方

Claude Codeで以下のように指示する:

```
「このスクショに合わせてUIを再現して」
「デザイン通りに見た目を合わせて」
「このモックアップをピクセル単位で再現して」
```

スキル発動後、以下を聞かれる:
1. 参照スクリーンショット（パス or 画像）
2. 対象URL
3. 開発サーバーが起動しているか

## フェーズ

1. **Phase 1** — 元スクショを具体的な数値で言語化
2. **Phase 1.5** — ピクセル色サンプリングでVLMが曖昧な色を数値確定
3. **Phase 2** — 言語化テキストだけを見て実装（画像は見ない）
4. **Phase 3** — Playwrightで現状スクショを撮影
5. **Phase 4** — 元スクショと現状スクショを並べて差分を言語化
6. **Phase 5** — 差分リストのHIGHから順に修正 → Phase 3に戻る

## 必要なもの

- **Playwright** — スクショ撮影に使用。未インストール時は案内が出る
- **sharp** — Phase 1.5のピクセル色サンプリングに使用
- **Web UI前提** — Playwrightによるブラウザスクショを使用するため、基本的にWeb UIが対象。ネイティブアプリの場合はスクショ撮影手段（`screencapture`コマンド、Appium等）を差し替えれば言語化ワークフロー自体は応用可能
- **5ループ制限** — 5回ループしても終わらない場合は一旦停止

## インストール

`SKILL.md`（英語版）または `SKILL.ja.md`（日本語版）をClaude Codeのスキルディレクトリにコピー:

```bash
# 個人（グローバル）スキル
mkdir -p ~/.claude/skills/ui-repro
cp SKILL.ja.md ~/.claude/skills/ui-repro/SKILL.md

# プロジェクトスキル
mkdir -p .claude/skills/ui-repro
cp SKILL.ja.md .claude/skills/ui-repro/SKILL.md
```

インストール後、Claude Codeを再起動してください。
