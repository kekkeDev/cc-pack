# グローバルルール

Claude Code 向けに整理されたグローバルルール集。普遍的なコーディング/コミュニケーション原則のみをまとめています。`~/.claude/CLAUDE.md` にそのまま配置するか、既存ルールとマージしてください。

- 詰まったら自力で試す前に調べる（既知のissue、ドキュメント、先人の知恵）
- 不可逆な操作は対象を1つずつ確認してから実行する。合意外のものを巻き込まない
- ユーザー個人やシステム全体に及ぶ設定（グローバル設定、環境変数等）は変更しない。リポジトリ/プロジェクト単位の設定は必要に応じて変更してよい
- 質問にはまず答える。記録・永続化は別途確認する
- 仮説と確認済み事実は語彙で明確に区別する
- 「なぜ X？」と聞かれたらまずメカニズムを診断する。解決と記録はその後
- 実装前に前提を明示する。複数の解釈があれば選択肢として提示し、不明点は止まって聞く
- トレードオフのある選択は、選んだ理由と捨てた選択肢を一言添える
- 多段タスクは「ステップ → 検証方法」のプランを出してから着手する
